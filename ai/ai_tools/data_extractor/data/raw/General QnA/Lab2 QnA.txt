Q:
Would the testing table for the C version and Java version be the same?
How about the algorithm, would they be different?

A:
The application test table would be the same as the functionality is the same independent of implementation (programming) language. 

So you use the same application test to test both the structured design version implemented in C and the OO version. Note that the OO version is **optional** depending on how much time you have. You can use C++ (preferred) instead of Java for the OO version.

As a minimum, you will submit the structured version implemented in C, not C++ (know the difference between the 2).

Improve adherence to the Single Responsibility principle.

The high level algorithm would be the same, again independent of programming language. Should not be more than half a page, and definitely no longer than a page.

Follow the algorithmic approach shown in the reference book Introduction to Algorithms.  As an example see Section 10.1 Stacks and Queues (you would have encountered these in ict167). Figure 10.1 in Thomas H. Cormen, et al. Introduction to Algorithms. The MIT Press, 2009. (access via My unit readings) shows the algorithms and it is easy to see with one look that the algorithm is constant time O(1). That is how you write algorithms. It makes easy to understand the gist of what is to be done.

Do not use verbose "essay writing" algorithms. Algorithms must be concise and independent of programming language. It has to be sufficiently abstract to capture the essence of the approach. Use meaningful names, but do not make them too long.

Q.
I have read the file into my array. The data file has duplicate names according to the question. I can’t set the duplicate item in the array to null as it is a record. Is there any way to delete an element in array.

A.
You can’t actually “delete” an array element. You can overwrite it by shifting one record at a time. Or you can take the last record and overwrite the one to be deleted. In either case, reduce count by one.

But the problem is what happens if the same name occurs multiple times in the array? Would a better approach be to insert into the array only if the name is not found in the array, and, update the coin amount if the name is already in the array?
